# Book Graphics Style Reference

**Clean, modern, minimal re-illustration style for consistent book visuals**

A professional warm aesthetic combining Notion's friendly minimalism, Stripe Docs' technical clarity, Apple Keynote's soft polish, and framework diagram structure.

---

## Core Philosophy: Warm Professional Minimalism

Every illustration balances **technical clarity** with **warm approachability** — clean vector aesthetics with soft gradients, generous spacing, and a slightly nerdy systems-thinking vibe.

**The Philosophy:** *"Technical warmth meets visual clarity."*
- Vector-like precision meets warm color harmony
- Soft gradients provide depth without clutter
- Clean strokes convey professionalism and trust
- Minimal compositions = Maximum clarity
- Systems-thinking diagrams made beautiful

---

## The Book Graphics Look

### What We Want
- **Vector-like clean illustrations** — Smooth, professional, high-quality
- **Warm color palette** — Orange warmth balanced with cool blues
- **Soft gradients** — Subtle transitions for depth and dimension
- **Minimal compositions** — 2-5 elements maximum, generous breathing room
- **Light backgrounds** — White or light grey for clean, open feel
- **Technical clarity** — Framework diagrams, structured visuals
- **Soft shadows** — Subtle depth (10-20% opacity)
- **Consistent style** — Every image feels like the same book

### Reference Styles
- **Notion** — Clean, minimal, warm, friendly illustrations
- **Stripe Documentation** — Technical diagrams with professional polish
- **Apple Keynote** — Soft gradients, clean shadows, modern aesthetic
- **Framework Diagrams** — Structured, systems-thinking, nerdy clarity
- **Figma/FigJam** — Clean vector work with warm personality

### What to AVOID
- Hand-drawn sketchy styles (no Excalidraw aesthetic)
- Dark or black backgrounds (use light backgrounds)
- Photorealistic images or textures
- Noisy patterns or excessive detail
- Cluttered compositions with too many elements
- Harsh neon colors or extreme saturation
- Rough edges or imperfect strokes
- Complex gradients everywhere (keep them subtle)

## Color System

### Backgrounds (Light & Clean)
```
Pure White          #FFFFFF   (primary - clean, open, professional)
Light Grey          #F8F9FA   (subtle variation for depth)
Soft Beige          #FAFAF8   (warm alternative background)
Very Light Blue     #F0F8FF   (optional cool background)
```

**Background Usage:**
- Default to pure white (#FFFFFF) for maximum clarity
- Use light grey (#F8F9FA) for subtle depth or sections
- Soft beige (#FAFAF8) for warmer feel when appropriate
- Always light, never dark

### Primary Accent: Warm Orange
```
Warm Orange         #FF6B35   (primary accent - energy, warmth, focus)
Soft Orange         #FF8C61   (lighter variation - softer areas)
Deep Orange         #E85D30   (darker variation - emphasis)
Coral Tint          #FFB399   (very light - backgrounds, fills)
```

**Orange Usage Guidelines:**
- Primary accent color (20-30% of colored elements)
- Use for focal points, key elements, emphasis
- Gradients from soft → warm → deep for depth
- Coral tint for subtle fills and backgrounds

### Secondary Accents: Blues (Technical)
```
Light Blue          #5AB9EA   (friendly, approachable technical)
Medium Blue         #3A8FCC   (professional, trustworthy)
Dark Blue           #2E5F8A   (depth, sophistication)
Navy Blue           #1E3A5F   (strong text, emphasis)
Pale Blue           #D4EBFA   (very light - backgrounds, subtle fills)
```

**Blue Usage Guidelines:**
- Secondary accent (15-25% of colored elements)
- Use for technical elements, structure, frameworks
- Medium blue for main technical components
- Dark/navy for text and strong outlines
- Pale blue for subtle fills and zones

### Neutrals (Structure & Text)
```
Pure Black          #000000   (primary text, sharp outlines)
Dark Grey           #4A4A4A   (secondary text, labels)
Medium Grey         #9B9B9B   (subtle elements, annotations)
Light Grey          #E0E0E0   (borders, dividers, structure)
Very Light Grey     #F5F5F5   (subtle backgrounds, zones)
```

**Neutral Usage Guidelines:**
- Black for primary text and key outlines
- Dark grey for secondary information
- Medium grey for subtle annotations
- Light grey for borders and structure
- Very light grey for zone backgrounds

## Color Hierarchy & Usage

**Color Distribution (ideal proportions):**
1. **White/Light Backgrounds** — 40-50% (generous negative space)
2. **Black/Grey Elements** — 20-30% (text, structure, outlines)
3. **Warm Orange Accents** — 15-20% (primary focus, warmth)
4. **Blue Accents** — 10-15% (technical elements, structure)
5. **Gradient Transitions** — 5-10% (depth, dimension)

**Combining Colors:**
- Orange + Light Blue = Warm professional technical
- Orange + White = Clean energetic focus
- Blues + Grey = Technical clarity
- All colors + White background = Clean modern minimal

## Gradient Specifications

### Soft Gradient Rules
Gradients should be **subtle and professional**, never overwhelming:

**Linear Gradients:**
- 2-color max (e.g., Soft Orange → Warm Orange)
- Gentle transition (30-40% gradient spread)
- Use for depth on primary elements
- Direction: top-to-bottom or left-to-right

**Radial Gradients:**
- Center-to-edge for spotlight effect
- Very subtle (e.g., White → Very Light Grey)
- Use sparingly for focal emphasis
- Opacity: 40-60% for background glows

**Example Gradient Combinations:**
```
Orange Depth:    #FF8C61 → #FF6B35 → #E85D30 (light to deep)
Blue Technical:  #5AB9EA → #3A8FCC (friendly to professional)
Subtle Glow:     #FFB399 (40% opacity) → transparent (radial)
Background:      #FFFFFF → #F8F9FA (barely visible)
```

**Gradient Don'ts:**
- No rainbow or multi-color gradients
- No harsh color transitions
- No gradients on everything (selective use)
- No complex mesh gradients

## Shadow & Depth System

### Soft Shadow Specifications
Shadows provide subtle depth without clutter:

**Element Shadows (for shapes, boxes, components):**
```
Soft Shadow:
  offset-x: 0-2px
  offset-y: 2-4px
  blur: 8-12px
  color: #000000
  opacity: 10-15%
```

**Elevated Elements (important components):**
```
Elevated Shadow:
  offset-x: 0-4px
  offset-y: 4-8px
  blur: 16-20px
  color: #000000
  opacity: 15-20%
```

**Subtle Depth (background elements):**
```
Minimal Shadow:
  offset-x: 0px
  offset-y: 1-2px
  blur: 4-6px
  color: #000000
  opacity: 8-12%
```

**Shadow Usage Guidelines:**
- Use for depth, not decoration
- Subtle and consistent throughout
- Lighter shadows on smaller elements
- Stronger shadows on focal points
- Always soft blur, never sharp edges

## Shape & Stroke Rules

### Vector-Like Clean Strokes
All lines and shapes should be **smooth, professional, and clean**:

**Stroke Characteristics:**
1. **Smooth curves** — Professional Bézier curves, no wobble
2. **Consistent weight** — 2-4px for main elements, 1-2px for details
3. **Rounded corners** — 4-8px radius for soft professional feel
4. **Clean edges** — No rough or hand-drawn aesthetic
5. **Precise alignment** — Grid-aligned, structured placement

**Shape Guidelines:**
- Rectangles: 4-8px corner radius (soft, not harsh)
- Circles: Perfect, smooth, no imperfections
- Arrows: Clean, 2-3px stroke, rounded ends
- Lines: Straight or smooth curves, consistent weight
- Icons: Minimal, 2px stroke, rounded style

### Line Styles
```
Primary Lines:      2-4px weight, #000000 or #4A4A4A
Secondary Lines:    1-2px weight, #9B9B9B or #E0E0E0
Accent Lines:       2-3px weight, #FF6B35 or #3A8FCC
Dividers:           1px weight, #E0E0E0
Connectors:         2px weight with arrow heads
```

## Composition Guidelines

### The Rule of Minimal
Every illustration follows strict minimalism:

**Element Limits:**
- **2-5 key components** maximum per image
- Each component should have clear purpose
- Remove anything that doesn't add value
- If it can be simplified, simplify it

**Spacing Requirements:**
- **30-40% white space** minimum
- Generous padding around all elements (20-40px)
- Clear separation between components
- Visual breathing room = clarity

**Layout Principles:**
1. **Grid-based structure** — Invisible grid for alignment
2. **Asymmetric balance** — Dynamic but stable
3. **Clear hierarchy** — Primary → Secondary → Tertiary
4. **Visual flow** — Guide eye through composition
5. **Focal point** — One clear area of emphasis

### Composition Types

**Framework Diagrams:**
- 2x2 matrices, flowcharts, decision trees
- Clean boxes with soft shadows
- Connecting lines with arrows
- Labels in dark grey (#4A4A4A)
- Orange accents on key decision points

**Conceptual Illustrations:**
- Abstract representation of ideas
- 3-5 shapes/elements maximum
- Soft gradients for depth
- Blue for technical, orange for human/warm
- Generous negative space

**Process Diagrams:**
- Step-by-step visual flows
- Numbered circles or boxes
- Directional arrows showing flow
- Orange for start/key steps
- Blue for process/technical steps

## Typography Guidelines

### When Text Is Needed
Typography should be **clean, readable, and minimal**:

**Font Recommendations:**
- **Sans-serif only** — Modern, clean, professional
- **Suggestions:** Inter, SF Pro, Helvetica, Arial
- **Weights:** Regular (400), Medium (500), Semibold (600)

**Text Hierarchy:**
```
Main Headlines:     24-32px, Semibold (600), #000000
Section Labels:     18-24px, Medium (500), #4A4A4A
Body Text:          14-16px, Regular (400), #4A4A4A
Small Annotations:  12-14px, Regular (400), #9B9B9B
```

**Text Placement:**
- Align to grid (left, center, or right — consistent)
- Generous spacing around text (12-20px padding)
- High contrast for readability
- No decorative fonts or scripts

**Text Colors:**
- Primary: Black (#000000)
- Secondary: Dark Grey (#4A4A4A)
- Annotations: Medium Grey (#9B9B9B)
- Accent: Warm Orange (#FF6B35) or Medium Blue (#3A8FCC)

## Lighting & Effects

### Subtle Professional Effects
Keep effects minimal and purposeful:

**Allowed Effects:**
1. **Soft shadows** — Depth and elevation (see Shadow System)
2. **Gentle gradients** — Dimension and focus (see Gradients)
3. **Subtle glows** — Emphasis on key elements (20-30% opacity)
4. **Transparency** — Layering and hierarchy (60-80% opacity)

**Glow Effects (Sparingly):**
```
Orange Glow (emphasis):
  color: #FF6B35
  opacity: 20-25%
  blur: 12-16px
  spread: 2-4px

Blue Glow (technical):
  color: #3A8FCC
  opacity: 15-20%
  blur: 10-14px
  spread: 2-4px
```

**Effects to AVOID:**
- Inner shadows (unless very subtle)
- Harsh highlights
- Texture overlays
- Noise or grain
- Complex lighting scenarios
- Excessive glows or blooms

## Image Types & Specifications

### Diagram Types

**Framework Diagrams (2x2, matrices):**
- Clean boxes with 8px rounded corners
- Soft shadows for depth
- Labels in dark grey
- Orange for high-priority quadrants
- Blue for analytical quadrants
- White/light grey backgrounds per section

**Flowcharts & Decision Trees:**
- Rounded rectangles (8px radius)
- Smooth connecting arrows (2-3px)
- Orange for decisions/key nodes
- Blue for processes
- Grey for standard nodes
- Clear directional flow

**Conceptual Illustrations:**
- Abstract shapes representing ideas
- 3-5 elements maximum
- Soft gradients on main shapes
- Floating composition with shadows
- Generous white space

**Process Diagrams:**
- Numbered steps (circles or rounded boxes)
- Sequential flow left-to-right or top-to-bottom
- Connecting arrows between steps
- Orange for start/key steps
- Blue for execution steps

### Technical Specifications

**Resolution & Quality:**
- **Minimum:** 2K resolution (2048px wide)
- **Preferred:** 4K for print quality
- **Format:** PNG with transparency or SVG for scalability
- **Color Space:** sRGB
- **Anti-aliasing:** Full anti-aliasing for smooth edges

**Aspect Ratios:**
- **16:9** — Landscape diagrams, presentations
- **4:3** — Standard illustrations, balanced compositions
- **1:1** — Icons, square compositions
- **3:2** — Portrait-friendly illustrations

**Export Settings:**
- PNG: 24-bit color + alpha channel
- SVG: Clean code, no unnecessary groups
- Quality: Maximum detail, no compression artifacts
- Background: Transparent or white (specified per use)

## Consistency Enforcement

### Maintaining Style Across Book

**Every image generation must:**

1. **Reference this style guide** explicitly
2. **Use exact color hex codes** from this document
3. **Apply consistent shadow specifications**
4. **Follow composition minimalism** (2-5 elements)
5. **Maintain white/light backgrounds** always
6. **Use soft gradients** sparingly and consistently
7. **Keep stroke weights** uniform (2-4px main elements)
8. **Apply negative prompts** for every generation

**Test for consistency:**
- Place multiple images side-by-side
- Check color palette harmony
- Verify similar shadow/gradient treatment
- Ensure uniform spacing and composition style
- Confirm same professional polish level

**If an image doesn't match:**
- Regenerate with stricter style enforcement
- Reference specific section of this guide
- Apply negative prompts more aggressively
- Increase style weight/emphasis in generation
